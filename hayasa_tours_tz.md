## 1. Общая информация

### Мобильное приложение
| Параметр | Значение |
|---|---|
| Платформы | iOS + Android |
| Фреймворк | React Native (Expo) |
| Карты | Yandex Maps SDK |
| Уведомления | Expo Push Notifications |

### Веб-версия
| Параметр | Значение |
|---|---|
| Платформа | Браузер (Desktop + Tablet + Mobile) |
| Фреймворк | Next.js 14 (React) |
| Хостинг | Vercel |
| Домен | hayasatours.com |
| Карты | Yandex Maps JS API |

### Общее для обоих продуктов
| Параметр | Значение |
|---|---|
| Бэкенд | Supabase (PostgreSQL + Auth + Storage) |
| Языки интерфейса | Армянский 🇦🇲 / Русский 🇷🇺 / Английский 🇬🇧 |
| Оплата | Офлайн (предоплата по договорённости) |
| Администратор | Один (владелец) |
| База данных | Единая — один Supabase для веба и приложения |

```
Supabase (единая БД)
    ├── hayasatours.com  (Next.js — веб)
    └── Hayasa App       (Expo — iOS + Android)
```

## 3. Технический стек

### Мобильное приложение (Expo)
- **Expo SDK** — сборка и публикация iOS/Android
- **Expo Router** — навигация между экранами
- **React Native** — UI компоненты
- **i18n-js + expo-localization** — мультиязычность
- **Yandex MapKit SDK** — карта с маршрутами и маркерами
- **Expo Push Notifications** — пуш-уведомления

### Веб-версия (Next.js)
- **Next.js 14** — App Router, SSR для SEO
- **Tailwind CSS** — стилизация
- **next-intl** — мультиязычность (hy/ru/en)
- **Yandex Maps JS API** — карта в браузере
- **Vercel** — хостинг и деплой

### Бэкенд (общий — Supabase)
- **PostgreSQL** — база данных
- **Supabase Auth** — аутентификация (Google, Apple, Email)
- **Supabase Storage** — хранилище фотографий
- **Supabase Realtime** — обновление статусов бронирований

## 4. Структура базы данных (Supabase)

### Таблица: `tours`
```
id              UUID PRIMARY KEY
title_hy        TEXT          — Название на армянском
title_ru        TEXT          — Название на русском
title_en        TEXT          — Название на английском
description_hy  TEXT
description_ru  TEXT
description_en  TEXT
category        TEXT          — 'classic' | 'gastro' | 'premium' | 'group'
date_start      TIMESTAMP     — Дата и время начала тура
price           INTEGER       — Цена в AMD
currency        TEXT          — 'AMD'
max_seats       INTEGER       — Максимум мест
booked_seats    INTEGER       — Забронировано мест
language        TEXT          — 'hy' | 'ru' | 'en' | 'all'
cover_image_url TEXT          — URL обложки тура
is_active       BOOLEAN       — Показывать ли тур клиентам
created_at      TIMESTAMP
```

### Таблица: `stops` (точки маршрута)
```
id              UUID PRIMARY KEY
tour_id         UUID REFERENCES tours(id)
order_index     INTEGER       — Порядок точки в маршруте
name_hy         TEXT
name_ru         TEXT
name_en         TEXT
description_hy  TEXT
description_ru  TEXT
description_en  TEXT
latitude        DECIMAL       — Широта
longitude       DECIMAL       — Долгота
created_at      TIMESTAMP
```

### Таблица: `stop_photos`
```
id              UUID PRIMARY KEY
stop_id         UUID REFERENCES stops(id)
photo_url       TEXT          — URL из Supabase Storage
order_index     INTEGER
```

### Таблица: `bookings` (бронирования)
```
id              UUID PRIMARY KEY
tour_id         UUID REFERENCES tours(id)
user_id         UUID REFERENCES auth.users(id)
seats           INTEGER       — Количество мест
status          TEXT          — 'pending' | 'confirmed' | 'paid' | 'cancelled'
full_name       TEXT
phone           TEXT
notes           TEXT          — Комментарий клиента
source          TEXT          — 'web' | 'app' — откуда пришёл клиент
created_at      TIMESTAMP
updated_at      TIMESTAMP
```

### Таблица: `profiles` (профили пользователей)
```
id              UUID REFERENCES auth.users(id)
first_name      TEXT
last_name       TEXT
phone           TEXT
avatar_url      TEXT
preferred_lang  TEXT          — 'hy' | 'ru' | 'en'
created_at      TIMESTAMP
```

## 5. Мобильное приложение — Экраны

### 5.1 Запуск приложения
**Сплэш-экран** → **Онбординг (3 слайда)** → **Выбор языка** → **Главная**

Онбординг показывается только при первом запуске. Слайды:
1. "Откройте Армению" — краткое описание приложения
2. "Маршруты и карты" — о функциях
3. "Бронируйте туры" — призыв к действию + кнопка "Начать"

---

### 5.2 Главный экран (Home)

**Верхняя часть:**
- Логотип Hayasa Tours
- Переключатель языка (🇦🇲 / 🇷🇺 / 🇬🇧) — сверху справа

**Баннер (ближайший тур):**
- Большое фото тура на всю ширину
- Поверх: название тура
- Обратный отсчёт: ДНИ : ЧАСЫ : МИН : СЕК (`#FFE20B`, JetBrains Mono)
- Кнопка "Подробнее"

**Список остальных туров (карточки):**
- Фото + Название + Дата + Обратный отсчёт
- Количество свободных мест (если мало — красный индикатор)

---

### 5.3 Экран поиска (Search)

**Строка поиска:** по названию тура (в реальном времени)

**Фильтры (горизонтальные чипы):**
- По дате: Этот месяц / Следующий месяц / Все
- По месту: выпадающий список (Гарни, Севан, Татев, Гегард и др.)
- По языку тура: 🇦🇲 / 🇷🇺 / 🇬🇧

---

### 5.4 Экран тура (Tour Detail)

**Галерея:** горизонтальная прокрутка фотографий (fullscreen по тапу)

**Информация:** название, дата, язык, места, цена, описание

**Маршрут (Roadmap):** вертикальная временная линия:
```
●── Точка 1: Гарни
│   Описание места...
│   [Фото 1] [Фото 2]
│
●── Точка 2: Гегард
│   Описание места...
│
●── Точка 3: Хор Вирап
    Описание места...
```

**Карта (Yandex Maps):**
- Пронумерованные маркеры
- Пунктирная линия маршрута
- Тап на маркер → название точки

**Кнопка:** "Забронировать" → авторизация → форма

---

### 5.5 Экран бронирования (Booking)

**Поля:** Имя + Фамилия, Телефон, Количество мест (+/-), Комментарий

**Итог:** Тур + Дата + Кол-во мест × цена = сумма

**Кнопка:** "Отправить заявку" → статус "Ожидание" + пуш администратору

---

### 5.6 Экран "Мои туры" (My Tours)

**Табы:** Предстоящие / Прошедшие

Каждая карточка: фото, название, дата, обратный отсчёт, статус бронирования

---

### 5.7 Экран профиля (Profile)

- Фото + имя + email
- Телефон
- Выбор языка (Armenian / Русский / English)
- История бронирований
- WhatsApp + Telegram кнопки
- Выйти из аккаунта

---

### 5.8 Экран входа/регистрации (Auth)

- Google / Apple / Email + пароль
- Гость может смотреть туры, бронирование требует авторизации

---

### 5.9 Навигация (мобильное приложение)
```
Нижняя панель
├── 🏠 Главная      → Список туров + баннер
├── 🔍 Поиск        → Фильтры + поиск
├── 📋 Мои туры     → Забронированные туры
└── 👤 Профиль      → Данные + язык + связь

Отдельные экраны:
├── Детали тура     → Галерея + roadmap + карта
├── Бронирование    → Форма заявки
├── Вход/Регистрация
└── Админ-панель
```

### 6.2 Главная страница (/)

**Navbar (sticky):**
- Логотип слева

---

### 6.4 Страница тура (/tours/[id])

**Правая колонка sticky (desktop):**
- Карточка: название, дата, цена, свободные места

**На мобильном:** sticky кнопка "Забронировать" внизу экрана

### 6.7 Профиль (/profile)

## 8. Админ-панель

Доступна в мобильном приложении. Один администратор (владелец).

### 8.1 Управление турами
- Список туров (активные / архив)
- Добавить / редактировать / удалить тур
- Поля: название (3 языка), дата, описание, цена, макс. мест, фото

### 8.2 Управление точками маршрута
- Добавить точку: название (3 языка), описание, координаты, порядок
- Загрузка фотографий → Supabase Storage
- Drag & drop порядок точек

### 8.3 Управление бронированиями
- Список заявок с фильтром по туру и статусу
- Смена статуса → автоматический пуш клиенту
- Контакт клиента: телефон, WhatsApp

## 9. Мультиязычность

Весь интерфейс и контент туров на 3 языках. Переключатель работает мгновенно.

| Продукт | Библиотека |
|---|---|
| Мобильное приложение | i18n-js + expo-localization |
| Веб-версия | next-intl |

Контент туров хранится в Supabase отдельными полями (`title_hy`, `title_ru`, `title_en`).

## 10. Этапы разработки

### Этап 1 — Мобильный MVP (1–2 месяца)
- Supabase: БД, авторизация, storage
- Главный экран + список туров
- Детали тура + roadmap
- Авторизация (Google + Email)
- Форма бронирования
- Экран "Мои туры"

### Этап 2 — Карты и контент (1 месяц)
- Yandex MapKit (мобильное)
- Фотогалерея на точках маршрута
- Мультиязычность

### Этап 3 — Уведомления и Admin (1 месяц)
- Expo Push Notifications
- Планировщик уведомлений
- Полная Админ-панель

### Этап 4 — Веб-версия Next.js (1–2 месяца)
- Настройка Next.js + Tailwind + Supabase
- Главная + страница туров + детали тура
- Бронирование через веб
- Мои туры + профиль (веб)
- Адаптивность (desktop / tablet / mobile)
- SEO (meta теги, sitemap, robots.txt)
- Деплой на Vercel → hayasatours.com

### Этап 5 — Полировка и публикация (2 недели)
- Тестирование iOS + Android + все браузеры
- Публикация App Store + Google Play
- Финальный деплой веба
